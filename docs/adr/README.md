# ADR (Architecture Decision Records)

Objetivo: registar decisões importantes com contexto e tradeoffs.

Formato:
- Um ficheiro por decisão: `NNNN-titulo-curto.md`
- Secções recomendadas: Contexto, Decisão, Alternativas, Consequências, Notas.

Quando criar ADR:
- Mudar stack/framework
- Definir modelo de auth/pagamentos
- Definir regras base de dados (ex.: soft delete, multi-tenancy)
- Definir padrões de integração com datasets externos
